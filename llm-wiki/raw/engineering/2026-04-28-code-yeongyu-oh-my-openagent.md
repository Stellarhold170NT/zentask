# Oh-My-OpenAgent (OmO) - Multi-model Orchestration for OpenCode

Oh-My-OpenAgent (formerly oh-my-opencode) is a specialized harness for the OpenCode terminal agent, designed to transform a single AI agent into a coordinated team of specialized units.

### Key Concepts
OmO handles parallel execution, model routing, and manages a team of agents (e.g., Sisyphus for long-running tasks, Prometheus for reasoning).

### Main Features
- **Orchestration**: Manages multiple models and sub-agents.
- **ultrawork (ulw)**: The primary command to trigger autonomous exploration, planning, and execution.
- **Configurable**: Uses JSONC for settings (`oh-my-openagent.jsonc`).

### Connection to GSD
There is no direct technical link between GSD and OmO. They are independent ecosystems developed by different creators. Both address the challenge of making AI agents more reliable and context-aware for complex engineering tasks.
- **OmO** focuses on OpenCode orchestration.
- **GSD** focuses on Claude Code context engineering and spec-driven workflow.
